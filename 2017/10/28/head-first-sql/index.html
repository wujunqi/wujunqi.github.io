数据和表

  • 数据库是保存表和其他相关SQL结构的容器
  • 数据库内的信息组成了表
  • 表的行包含了表中的某个对象的所有信息。
  • 字段（field）也常用来代称列，记录与行交替使用
  • RDBMS:relational database management system，关系型数据库管理系统
  • 创建数据库：

    • USE 数据库名
  • 命令大写是良好的SQL编程习惯

select 语句

  1. SELECT * FROM 表名

  2. 可以使用WHERE 字句缩小查询范围：SELECT * FROM 表名 WHERE name = ‘wujunqi’;

  1. 如果在单引号数值内部包含单引号，请使用反斜杠\， 如’wujunqi\’ wife’; 还有另外一种方法，就是输入两个‘’表示转义

  1. 请不要使用双引号””代替’’，因为日后mysql还要搭配php语言，而在php语言中，被双引号包围的语句才是SQL语句，如果又在SQL语句中又包含“”，则在双引号中的sql语句中的值不会当成sql语句的值。

  2. SELECT 可以采用特定列来限定查询的结果，只要指定你要查询的列即可，如：SELECT name, sex FROM 表名 WHERE name = ‘wujunqi’;

  3. AND 和 OR 操作，= <>:不等于号，>,<,>=,<=， 字符表示按字母排序。

  4. 选择出某个值是否为NULL，可以用IS NULL表示， 例如： SELECT * FROM 表名 WHERE name IS NULL;

  5. LIKE :查找字符中又其的记录，可以搭配使用 %(任意数量未知字符的替身)， 和_(一个未知字符的替身)，如：SELECT * FROM test WHERE name LIKE ‘%jun’;

  1. BETWEEN AND:等价于：>= and <= 如：SELECT * FROM test WHERE BETWEEN age BETWEEN 18 AND 25;

  1. IN: 在IN后面加上括号围起的集合，只有列值匹配集合中的任意值，就返回该记录值，NOT 能反转查询的值，即找出不在集合中的记录。如：

  1. NOT： 可以和LIKE 和 BETWEEN 配合使用，但必须放在WHERE 的后面(IN 是一个例外)，当和AND和OR一起使用时，直接放在其后面：

  1. DELETE: 删除表中的某一条记录，注意和DROP区别，DROP会删除表的结构。

  1. 慎重使用DELETE,确认自己加入了非常精确的WHERE字句。可以选出你真正想要删除的行，这时候可以先使用SELECT进行预先操作。

  2. UPDATE: 更新一列或者多列的值， 如：UPDATE test SET age = 22;（把表中age这一列中的所有年级都改为22）；

聪明的表设计

  1. 表都是关于关系的，对于设计表而言，我们就是要考虑列彼此之间如何产生关系、如何一起描述某项事务。 设计的方向取决于我们对表格的需求。创建表格是一般需要遵循：

  2. 原子性数据：
    原子是一块无法或者不应该分割的信息。对数据而言也是一样，当数据具有原子性（atomic），就表示已经分割至最小块，已经不能或者不应该再分割了。

  3. 原子性数据和你的表

  1. 原子性数据的规则：
    • 具有原子性数据的列中不会有多个类型相同的值。
    • 具有原子性数据的列中不会有多个存储同类型的列。

  1. 规范化：

    • 让数据具有原子性是创建一个规划化表格的第一步。
    • 规范化表格的优点：①规范化的表格没有重复数据，可以减少数据库的大小。②因为查找的数据较少，查找会很快。
    • 具有原子性数据只让我们的表规范了一半。完全的规范化表示我们处于第一范式(First Normal Form)的状态。简称1NF.
    • 要让表具有完全规范化状态，还需要： 每个数据行都必须有独一无二的识别项，人称主键(primary key ), 主键是表中的一列，它可以让每一条记录成为唯一的。

  1. 模式：
    用于表达数据库内的结构，包括行和列，还有各种他们之间相互连接的方式。
    对数据库内的数据描述（列和表），以及任何相关对象和各种连接方式的描述就称为SCHEMA，模式。
  1. 外键：
    外键是表中的某一列，它引用于（到）另一个表的主键。

  1. 外键约束：
    利用CREATE 或ALTER语句指定外键，创建在结构内的键被称为约束(constraint)。

  2. 引用完整性：
    插入外键列的值必须已经存在于父表的来源列中，这是引用完整性（referential integrity）；
